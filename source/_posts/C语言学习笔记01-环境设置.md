# C语言学习笔记01-环境设置
## 前言
C语言我相信对于每个学习编程的人来说应该都是第一个学习的语言吧，记得自己上高中的时候不喜欢数学，数学课上就偷偷看谭浩强的《C语言程序设计》。对我来说已经不是一门陌生的语言，但是因为时间太过久远，现在就是一个理论上是复习，但是依旧可以学到新东西的一个状态，这一系列的博客就是记录自己的一个学习过程，那就少啰嗦，看东西。
## 环境准备
如果想在自己的电脑上安装C语言的运行环境，那么请保证自己电脑上要有以下两款可用的软件**文本编辑器**和**C编译器**
> 其实现在不用这么麻烦，直接去装一个C的集成开发环境就好了，不过这种方式可以更加清晰的帮你认识C语言的运行过程，可以试试。
### 文本编辑器
文本编辑器用于编写程序的源代码，其实Windows系统自带的记事本 和macOS下的文本编辑app就是一个简单文本编辑器，但是因为功能过于简单所以并不推荐，更推荐像Atom，Visual Studio Code等一些可以安装一些语言插件的文本编辑器，这些软件用起来可能更顺手，友好一些。
### 编译器
这里指的编译器不是大家所熟知的Dev C++，Visual Studio 等软件，他们真正的身份是**集成开发环境**，是为了方便开发者编写，运行，测试而将文本编辑器和编译器以及其他的一些开发测试工具打包在一起的一个开发环境
最常用的免费可用的编译器是 GNU 的 C/C++ 编译器**gcc**。编译器用于将源代码编译成为最终的可执行程序
## 安装过程
接下来将介绍如何在你的操作系统上安装C的开发环境
### Unix/Linux/macOS系统
Linux或Unix一般都在系统中集成了C编译器，如gcc,clang,g++等，请在终端中使用下面的命令来检查您的系统上是否安装了gcc：  
`$gcc -v`
如果你的计算机上已经安装了gcc 那么则会显示以下信息或类似的信息
```shell
Configured with: --prefix=/Library/Developer/CommandLineTools/usr --with-gxx-include-dir=/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/usr/include/c++/4.2.1
Apple clang version 11.0.3 (clang-1103.0.32.62)
Target: x86_64-apple-darwin20.0.0
Thread model: posix
InstalledDir: /Library/Developer/CommandLineTools/usr/bin
```
> 我用的是macOS，macOS的内核使用的是Unix BSD，所以可以视为类Unix系统  。
> macOS用户怕麻烦可以直接安装Xcode。

### Windows系统
在 Windows 上安装 GCC需要安装 MinGW。为了安装 MinGW，请访问 MinGW 的主页 www.mingw.org，进入 MinGW 下载页面，下载最新版本的 MinGW 安装程序，当安装 MinGW 时，您至少要安装 gcc-core、gcc-g++、binutils 和 MinGW runtime，但是一般情况下都会安装更多其他的项。安装完成后请将安装好的工具添加到系统环境变量PATH中，以便在命令行工具中使用这些工具。具体操作请自行百度。
> 如果实在怕麻烦可以直接安装Visual Studio的社区版本，只是受微软的影响，他的C“血统”可能不是那么纯正。

以上就是关于C语言的环境配置工作。
